递归简论

当一个函数用它自己来定义时，就称为递归。简单来讲，就是函数内部调用了函数自己本身。

递归法则

  1. 基准情形。必须总有某些基准情形，这样才能求解。
  2. 不断推进。对于需要递归求解的情形，每一次递归都要朝着基准情形推进。
  3. 设计法则。假设所有的递归调用都能运行。
  4. 合成效益法则。在求解一个问题的同一实例时，切勿在不同的递归调用中做重复性工作。

常用递归算法

1，斐波那契数列
2，幂运算（正整数幂),模拟pow方法（函数）
3，汉诺塔
4，分治算法求最大子序列
……..

斐波那契数列

根据斐波那契数列定义 fib（N） = fib(N-1) + fib(N-2), fib(1) = 1 ,fib(2) = 1;
可以很容易的到Fib那契数列的算法
  public Long fib(Long n){
        if(n == 1 || n== 2){  // 基准情形
            return 1L;
        }else {
            return fib(n-1) + fib(n-2); // 调用自身并且朝着基准情形推进，但是它做重复性工作
        }
    }

很显然上面的算法满足递归法则1，2，3 但是第四条却不满足，因为fib(n-2)会去做fib(n-1)
中的重复工作，所以这样的算法未必就是好的。

幂运算（正整数幂),模拟pow方法（函数）

 幂运算，10的3次方用C语言的函数可表示为Pow(10,3),在幂为正整数的条件下，可以用递归
 写出该算法

public long Pow(Long x, long N) {
    if (N == 0) {    // 基准情形
        return 1;
    }
    if (N == 1) {
        return x;   // 基准情形
    }
    if (N % 2 == 0) {
        return Pow(x * x, N / 2);  / 调用自身并且朝着基准情形推进
    } else {
        return Pow(x * x, N / 2) * x; / 调用自身并且朝着基准情形推进
    }
}

上面的算法满足递归的所有法则，所以它是一个比较好的算法。如果不用递归来模拟幂运算，用for循环
来写，我想那会是一个很不优美的写法。

参阅书籍

《数据结构与算法分析》  --  Mark Allen Weiss

结语

汉诺塔，分治算法求最大子序列还有N多递归算法，设计的十分精巧，甚至不用递归无法解决问题（汉诺塔很典型），
但递归不完全都是好的，递归算法设计起来相当费力，理解起来，也比较费力，适当的算法才是最好的。
